AI Agent

In the realm of artificial intelligence, an “agent” is a software or hardware entity capable of perceiving its environment and taking actions to achieve specific goals. AI Agent = LLM-based Autonomous Intelligent Entity (Large Language Model-based Autonomous Agents). These agents may also be referred to as LLM Agents, Autonomous Agents, or Autonomous Intelligent Entities.

An Intelligent Agent (IA) perceives its environment, autonomously takes actions to achieve its goals, and can improve its performance through learning or knowledge acquisition. Intelligent agents can range in complexity from simple tools to full collaborative partners, even fully autonomous replacements. Goal-oriented behavior is considered the essence of intelligence in these agents.

An Autonomous Agent is a system capable of acting independently and purposefully within the real world. **Characteristics of Autonomous Agents **includes 1) Perception of the environment, 2) Autonomous decision-making, 3) Ability to act, 4) Clear goals and tasks, 5) Adaptability to the environment, 6) Learning capability.

1. Planning

  • Subgoal and Decomposition: The agent breaks down large, complex tasks into smaller, manageable subgoals, which helps it effectively handle tasks of greater complexity.
    • Chain of Thought (CoT) technique has become a standard prompting method to improve model performance on complex tasks. CoT involves breaking down tasks into sequential thinking steps, making it easier for the model to tackle intricate problems.
      • Zero-Shot CoT: The model attempts complex reasoning without needing prior examples or training on specific tasks.
      • Few-Shot CoT: The model uses a small number of examples to guide its reasoning process, enhancing its ability to understand and solve tasks.
    • Tree of Thoughts (ToT):
      Introduced by Yao et al. in 2023, ToT expands on CoT by exploring multiple potential reasoning paths at each step, creating a tree structure. This approach supports complex reasoning by evaluating multiple possible steps simultaneously.
      • Search Methods:
        The ToT method can utilize Breadth-First Search (BFS) or Depth-First Search (DFS), with each state evaluated by classifiers (through prompting) or majority voting.
  • Reflection and Refinement: The agent can engage in self-criticism and self-reflection, learning from past actions to improve future performance and ultimately deliver higher-quality results.
    • Techniques like ReAct (by Yao et al. in 2023) and Reflexion (by Shinn & Labash in 2023) enable agents to perform self-assessment, reflect on past actions, and refine their strategies based on feedback.

2. Memory

Memory in AI agents refers to the process of acquiring, storing, retaining, and retrieving information, and it is inspired by human memory systems.

  • Sensory Memory: This type of memory briefly retains sensory information right after a stimulus ends, typically lasting only a few seconds.
  • Short-Term Memory (STM) or Working Memory: Stores information that we are currently aware of, lasting around 20–30 seconds. Short-term memory allows the agent to engage in In-Context Learning (ICL), enabling it to learn within the given context.
  • Long-Term Memory: Long-term memory enables the agent to store and recall information over extended periods, typically by using external vector storage systems that facilitate fast information retrieval.
    • Types of Long-Term Memory:
      • Explicit/Declarative Memory: Conscious memory of facts and events.
        • Episodic Memory: Memory of personal experiences or events.
        • Semantic Memory: Memory of facts, concepts, and knowledge.
      • Implicit/Procedural Memory: Unconscious memory of skills and routines, such as riding a bicycle or typing on a keyboard.

Mapping AI Memory to Human Memory Types

  • Sensory Memory in AI is akin to embedded representations.
  • Short-Term Memory corresponds to the context or prompt used in LLMs (Large Language Models).
  • Long-Term Memory is implemented through vector storage for long-term information retention and retrieval.

3. Tools

Tool Usage: The agent learns to call external APIs to obtain additional information not stored in its model, such as real-time data, code execution capabilities, and access to proprietary information sources. This allows the agent to expand beyond its pre-existing knowledge base. For example, Function/Tool Calling.

4. Action

The Action component involves translating the agent’s decisions into tangible outcomes, whether that means executing specific tasks, interacting with its environment, or influencing its internal state.

  • Goal of Actions: Actions are directed toward achieving specified goals or objectives, such as:
    • Completing a task
    • Communicating information
    • Exploring and interacting with the environment
  • Types of Actions:
    • Using Memory: Actions based on information stored in memory.
    • Following a Plan: Actions guided by a pre-defined or dynamically generated plan.
  • Action Space:
    • Includes a range of external tools, APIs, databases, knowledge repositories, and external models that the agent can access as needed.
    • Internal Knowledge of LLM: Actions can also leverage the model’s pre-trained knowledge.

AI Agent Construction Strategies

If one aims to build an agent, it is essential to consider these core components in the system’s design. Although the underlying components are similar, each agent type emphasizes different aspects of functionality depending on the specific use case.

Categorization of AI Agent depends on how it’s constructed based on planning, memory, tools and action: LLM-Agent-Survey Github .

  1. Assistant Agents:

    • Example: AgentGPT, which leans toward assisting users in task management.

  2. Tool Agents:

    • Example: HuggingGPT, primarily focused on tool usage and integration.

  3. General Agents:

    • Example: AutoGPT, oriented towards general-purpose problem-solving across various contexts.

  4. Simulation Agents:

    • Example: Generative Agent, geared toward social simulation and multi-agent collaboration.

Agent - Reflection and ReAct

Reflection: Think -> Action -> Observation

Reflection in AI often refers to the process where an agent can analyze its own reasoning, actions, and the outcomes of those actions. It involves self-assessment and the ability to adjust future behavior based on past experiences. Key aspects include:

  • Self-awareness: The ability of an AI to monitor its internal states and processes.
  • Learning from Experience: Using previous interactions and outcomes to improve future decision-making.
  • Error Analysis: Identifying mistakes or suboptimal actions and understanding why they occurred.
  • Paper: Reflexion: Language Agents with Verbal Reinforcement Learning
  • Code: https://github.com/noahshinn/reflexion
  • Tutorial: https://www.promptingguide.ai/techniques/reflexion

ReAct: Reasoning -> Action

ReAct is a framework designed for AI agents that emphasizes the integration of reasoning and action in a dynamic environment. Key components include:

  • Dynamic Decision Making: AI agents utilize reasoning to evaluate situations and choose actions in real-time.
  • Interaction with the Environment: The ability to adapt and respond to changes in the environment while reasoning through potential outcomes.
  • Combining Knowledge: Leveraging both learned knowledge and reasoning capabilities to execute complex tasks or solve problems.
  • Paper: ReAct: Synergizing Reasoning and Acting in Language Models
  • ReAct Project: https://react-lm.github.io/
  • ReAct Repo: https://github.com/ysymyth/ReAct
  • ReAct in LangChain v0.1: https://python.langchain.com/v0.1/docs/modules/agents/agent_types/react/
  • Tutorial: https://www.promptingguide.ai/techniques/react

ReAct is also a type of Prompting Methods, others include original method, CoT, and Act.
